# 精修 Schema 操作 UX

## 为什么

Akt 已经有对象树、创建数据库、创建数据表、表结构查看和部分 alter-table 的基础规格，但这些能力仍偏工程拆分，缺少面向用户的完整操作节奏。

本变更围绕四条高频 schema 操作做一轮精细 UX 设计：

- 新建数据库。
- 修改数据库。
- 新建数据表。
- 修改数据表。

设计目标是对齐 Navicat 类数据库客户端的自然使用方式：对象树定位、右键或工具栏进入操作、结构化设计器分 tab 编辑、保存前可见变更摘要、执行后刷新对象树并保留上下文。

## 变更内容

- 新增 `schema-operations-navicat-ux` 能力，定义四类 schema 操作的 UX 行为。
- 为四条流程提供像素级工作台规格和逐步交互。
- 统一 schema 操作入口：对象树右键、顶部新建按钮、详情页动作条。
- 统一 create/modify 表单策略：结构化表单为主，变更计划、校验、风险和结果为审阅对象。
- 统一保存策略：普通创建直接执行；修改类操作先生成变更计划，再执行。
- 统一危险操作边界：删除、清空、字段删除、索引删除、约束删除必须确认。
- 统一执行反馈：结果、耗时、失败阶段、对象树刷新和可读恢复建议。

## 非目标

- 本变更不实现具体数据库驱动。
- 本变更不要求所有数据库支持所有字段选项。
- 本变更不把 MongoDB 强行纳入表设计器；MongoDB collection 设计会单独映射。
- 本变更不复制 Navicat 品牌、图标或专有视觉资产；对齐的是专业数据库客户端的操作节奏、信息架构、组件密度和状态规则。
- 本变更必须给出 Akt 自己的像素级布局规格，避免后续实现漂移。

## 影响

- 受影响 specs：`schema-operations-navicat-ux`
- 后续会影响现有 specs：`object-workbench`、`connection-manager`、`query-console`
- 后续会影响代码：对象树动作、数据库设计器、表设计器、变更计划、执行日志、状态栏
